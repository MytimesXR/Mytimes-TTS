# 安装与使用教程

当前公开版本支持 64 位 Windows。

## 一、选择正确文件

- 推荐：文件名包含 `Portable` 和 `x64` 的 EXE，无需安装，下载后直接运行。
- 备用：文件名包含 `x64` 的 Windows ZIP，先解压到普通文件夹，再运行其中的 `Mytimes TTS.exe`。

不要直接在 ZIP 压缩包预览窗口里运行程序。macOS 和 Android 尚未作为公开下载平台提供。

## 二、完成首次初始化

第一次打开应用时会出现一次性引导：

1. 阅读“填写正文 → 选择声音 → 生成与保存”的简单介绍。
2. 选择“公开版（推荐）”；该模式不会检测或读取任何组织内部配置。
3. 使用本机“文档\Mytimes-TTS-Data”，或选择自己的本机、移动硬盘、映射盘或可信网络目录。
4. 应用会确认所选目录当前可读写。
5. 应用检查当前目录中的 `settings.json` 和这台电脑可用的加密 API Key。
6. 缺少配置或 Key 时，引导完成后自动进入设置页。

初始化完成后，Windows 会在当前用户的应用数据目录中保存一个很小的 `storage-state.json`，只记录所选路径、运行模式和引导状态。以后直接打开 EXE；只有目录断开、移动或删除时才重新提示。可在“设置 → 数据目录”中选择、打开或恢复默认目录。

## 三、选择并配置语音引擎

1. 按 [MiMo API Key 获取与配置教程](MIMO_API_KEY_GUIDE.md) 获取自己的 Key。
2. 打开 Mytimes TTS，进入左侧“设置”。
3. 选择“标准 API”或“Token Plan”。
4. 保持对应的官方 Base URL，或填写经过确认的兼容服务地址。
5. 输入 API Key 后可直接点击“保存设置”，这个操作不会调用 MiMo。
6. 如需验证连接，点击“保存并测试（调用 API）”。应用会先弹出费用提醒；只有确认后才发送一条很短的测试请求，该请求仍可能计入 Token 或额度。

API Key 不在安装包内。每位用户首次使用时自行填写，应用会通过 Windows DPAPI 按电脑和 Windows 用户加密后保存在当前数据目录。

还可以在同一设置区切换到火山引擎、GPT-SoVITS 或 IndexTTS2。火山引擎连接测试可能产生少量用量；两个本地引擎的连接测试不调用 MiMo 或火山引擎。完整部署与字段说明见 [多语音引擎配置](MULTI_ENGINE_GUIDE.md)。

## 四、生成语音

### 预置音色

1. 打开“语音生成”。
2. 选择 MiMo 或火山引擎，再选择“预置音色”。
3. 选择一个音色并填写待朗读正文。
4. 可填写自然语言风格，或点击风格标签。
5. “规则整理”完全在本机执行，不调用任何模型。
6. “独立 LLM 润色”只优化风格描述，不修改正文；默认关闭，且不会使用 MiMo API Key。
7. 点击“生成语音”。

### 音色设计

1. 切换到“音色设计”。
2. 用年龄、音高、语速、情绪、质感、口音和使用场景等特征描述声音。
3. 填写正文并生成。

### 音色复刻

1. 切换到“音色复刻”。
2. 点击上传区域选择 WAV/MP3，或把文件拖入区域。
3. 只使用本人声音或已取得明确授权的声音。
4. 填写正文和演绎风格后生成。

GPT-SoVITS 还需要填写参考音频原文；IndexTTS2 可以使用风格文字或八维情绪向量。本地服务必须先在本机或可信局域网设备上启动。

## 五、播放与保存

- 点击播放键开始或暂停。
- 点击波形可跳到对应时间；按住并拖动可连续定位。
- 点击“保存音频”选择导出位置。
- “生成历史”保存最近的生成记录和应用自己的 WAV 副本。

## 六、可选：配置本地 llama.cpp 润色

不需要 AI 润色时保持默认的“仅规则整理”即可，生成语音功能不受影响。

1. 在本机启动 `llama-server`。官方服务默认监听 `127.0.0.1:8080`，并提供 OpenAI 兼容的 `/v1/chat/completions`。
2. 打开“设置 → 独立润色服务”。
3. 选择“llama.cpp / OpenAI 兼容接口”。
4. Base URL 填写 `http://127.0.0.1:8080/v1`。
5. 单模型服务可暂时使用默认模型名；使用路由或别名时，填写服务实际模型 ID。
6. 本地服务没有启用鉴权时，独立 API Key 留空；有鉴权时填写该服务自己的 Key。
7. 保存设置后返回语音页，点击“独立 LLM 润色”。

该接口与 MiMo TTS 配置完全分离，不会读取、复用或回退到 Xiaomi MiMo API Key。详见 [llama.cpp 官方服务器说明](https://github.com/ggml-org/llama.cpp/blob/master/tools/server/README.md)。

## 七、数据目录与网络存储

- 默认目录：当前 Windows 用户的“文档\Mytimes-TTS-Data”。
- Portable EXE 所在目录不会自动生成配置文件、初始化标记或数据文件夹。
- 自定义目录：进入“设置 → 数据目录”，点击“选择目录”，可选择本机磁盘、映射盘或 NAS 文件夹。
- 切换到空目录时，应用会复制当前设置、历史和生成音频，并自动修正历史文件路径。
- 目标目录已经有 Mytimes TTS 数据时，可以选择使用目标数据，或用当前数据覆盖同名文件。
- API Key 按电脑和 Windows 用户分别加密；另一台电脑使用同一 NAS 目录时，需要在那台电脑上重新填写 Key。
- 不建议多台电脑同时运行并写入同一个历史目录。
- 设置页的“重新演示首次引导”只重新打开引导界面，不会删除数据、设置、历史或 API Key。

从旧版升级时，应用会优先继承已经选择的数据目录。0.1.5 生成的两个 EXE 旁索引文件会在成功迁移到 Windows 用户索引后清理，但原数据目录、设置、历史和音频不会删除。

## 八、删除本机数据

- 在“生成历史”中删除单条记录，会删除应用保存的对应音频副本。
- “清空历史”会清理应用保存的历史记录和对应 WAV。
- 在“设置”中清空 API Key 并保存，可移除应用保存的 Key。
- 删除 Portable EXE 不会自动删除所选数据目录，也不会删除 Windows 用户目录中的路径索引。
- 删除本地 `storage-state.json` 只会让初始化引导下次重新出现，不会删除所选目录中的业务数据。

另存到其他目录的音频和原始参考音频不会被应用的历史清理功能删除。

## 九、常见问题

### 验证连接失败

确认服务类型、Base URL 和 Key 相互匹配，再检查网络、额度与 MiMo 控制台状态。完整排查见 [MiMo API Key 教程](MIMO_API_KEY_GUIDE.md)。

### Windows 出现 SmartScreen

当前构建可能尚未完成 Windows 代码签名。确认文件来自本仓库 Releases 后再运行；来源不明的文件不要打开。

### Portable 与 ZIP 有什么区别

Portable EXE 下载后可直接运行；ZIP 需要先完整解压。两者功能相同，优先选择 Portable EXE。

### 为什么没有 macOS 下载

macOS 仍处于未来适配和验证阶段，尚未作为当前公开支持平台发布。请不要把自动构建产生的预览文件当作正式版本分发。
