# 隐私说明

本说明描述 Mytimes TTS 应用本身的数据处理方式，不代表 Xiaomi MiMo 平台的隐私政策。

## 数据会发送到哪里

用户点击生成或验证连接时，应用会把完成当前操作所需的数据发送到用户配置的 API 地址：

- 预置音色：正文、风格描述、音色 ID 和生成参数。
- 音色设计：正文、音色特征描述和生成参数。
- 音色复刻：正文、风格描述、参考 WAV/MP3 和生成参数。
- AI 风格润色：只发送风格描述，不发送待朗读正文。
- 验证连接：发送一条很短的测试请求。

默认服务是 Xiaomi MiMo。填写自定义 Base URL 后，数据会发送到该自定义服务；使用前应确认服务提供方可信并阅读其隐私政策。

当前应用没有 MytimesXR 账号、云同步、广告 SDK 或主动遥测上报，不会把生成历史主动上传到 MytimesXR 服务器。

## 本机保存哪些数据

应用会在当前 Windows 账户的 Electron 用户数据目录中保存：

- 连接设置和加密后的 API Key。
- 生成历史元数据，包括正文、风格、模式、音色、生成时间和本地文件路径。
- 应用保存的 WAV 历史副本，最多保留最近 100 条。

API Key 使用 Electron `safeStorage` 调用 Windows DPAPI 对应的系统安全能力。它可以降低本机明文泄露风险，但不能抵御已经控制当前用户会话的恶意程序，因此仍应限制 Key 权限、额度并定期轮换。

参考音频会在生成请求中读取并发送。应用不会把用户选择的原文件另存为历史样本，也不会删除原文件。

## GitHub 与安装包不包含本机数据

公开源码和构建配置只收录项目白名单文件，不读取 Electron 用户数据目录。因此：

- 不包含用户已经填写的 API Key。
- 不包含 `settings.json`、`history.json`。
- 不包含参考音频、生成 WAV 或另存音频。
- 不包含个人目录、日志和签名凭据。
- 打包不会清空或修改当前电脑上的 Key 与历史。

当前 Windows 应用包只收录运行所需的 `electron/`、`app/` 和 `package.json`。

## 如何清除数据

- 删除单条历史：同时删除应用保存的对应 WAV 副本。
- 清空历史：删除应用保存的全部历史记录和对应 WAV。
- 在设置中清空 API Key 并保存：移除应用保存的 Key。
- 删除 Portable 程序不一定自动清除 Electron 用户数据目录；如需彻底清理，请先确认目录内容再手动删除。

应用不会删除用户另存到其他目录的音频，也不会删除最初选择的声音样本。

## 声音复刻

声音可能涉及个人信息、人格权、著作权、合同和平台规则。只能上传本人声音，或已经取得明确、有效授权的声音。应用内的授权确认只是操作提醒，不能替代真实授权和法律审核。

不得使用本应用从事冒充、诈骗、骚扰、未经同意的公开传播或其他违法、侵权活动。

## API Key 安全

- 每位用户使用自己有权使用的 Key。
- 不要在截图、录屏、Issue、聊天、日志或 Git 仓库中公开 Key。
- Key 泄露后，应立即在 Xiaomi MiMo 控制台重置或删除，并检查调用记录。
- 统一提供额度时，应使用受控后端代理，不能把共享 Key 内置进公开 EXE、APK 或网页。

## 联系与变更

安全问题请使用本仓库的 GitHub Security Advisory 私下报告；不要在公开 Issue 中发布真实 Key、用户正文或声音样本。

未来若增加账号、服务器代理、遥测、崩溃上报、云同步或移动端权限，本说明必须在发布前同步更新。
