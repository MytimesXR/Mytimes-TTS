# 隐私说明

本说明描述 Mytimes TTS 应用本身的数据处理方式，不代表 Xiaomi MiMo、火山引擎或本地模型项目的隐私政策。

## 数据会发送到哪里

用户点击生成或验证连接时，应用会把完成当前操作所需的数据发送到用户配置的 API 地址：

- 预置音色：正文、风格描述、音色 ID 和生成参数。
- 音色设计：正文、音色特征描述和生成参数。
- 音色复刻：正文、风格描述、参考 WAV/MP3 和生成参数。
- 本地规则整理：不发送任何数据，不调用模型。
- 独立 AI 风格润色：默认关闭；启用后只把风格描述发送到单独配置的 OpenAI 兼容地址，不发送待朗读正文。
- 验证 MiMo 或火山引擎连接：用户在费用提醒中确认后发送一条很短的测试请求。
- GPT-SoVITS：正文、参考原文和参考音频路径发送到用户配置的本地或局域网 API；参考音频会短暂写入 Windows 临时目录并在请求结束后删除。
- IndexTTS2：正文、参考音频和情绪控制参数发送到用户配置的本地网关。

TTS 默认服务是 Xiaomi MiMo。切换到火山引擎、本地模型或填写自定义地址后，语音相关数据会发送到当前选中的服务。独立润色接口与 TTS 接口完全分离，不会读取或回退使用任何 TTS Key；使用任何自定义服务前都应确认提供方可信并阅读其隐私政策。

当前应用没有账号系统、云同步、广告 SDK 或主动遥测上报，不会把生成历史主动上传到项目维护者的服务器。

## 数据保存在哪里

公开版首次启动不会检测组织内部目录。用户可以使用本机“文档\Mytimes-TTS-Data”，或选择其他本机磁盘、移动硬盘、映射盘、NAS 文件夹。所选目录保存：

- `settings.json`：各 TTS 引擎与独立润色服务的非敏感连接参数、模型名、超时和主题，不含明文 Key。
- `secrets/`：按电脑和 Windows 用户隔离的加密 MiMo Key、火山引擎 Access Token、可选本地服务 Key 与独立润色 Key。
- `history.json`：生成历史元数据，包括正文、风格、模式、音色、生成时间和本地文件路径。
- `generated/`：应用保存的 WAV 历史副本，最多保留最近 100 条。

API Key 使用 Electron `safeStorage` 调用 Windows DPAPI。加密结果只能由对应电脑和 Windows 用户正常解密，因此多台电脑共用 NAS 数据目录时，每台电脑需要分别填写自己的 Key。该机制不能抵御已经控制当前用户会话的恶意程序，仍应限制 Key 权限、额度并定期轮换。

参考音频会在生成请求中读取并发送。应用不会把用户选择的原文件另存为历史样本，也不会删除原文件。GPT-SoVITS 官方 API 需要文件路径，因此应用会在 Windows 临时目录创建一次性副本并在请求结束后删除；程序异常退出时应检查系统临时目录中的 Mytimes-TTS 子目录。

从旧版升级时，应用会继承已有数据位置。切换数据目录只复制或切换，不主动删除原数据目录。0.1.5 的两个 EXE 旁索引文件在成功导入新的 Windows 用户索引后会被清理，但不会删除原数据目录。

NAS 目录适合在多台电脑之间顺序使用。当前历史文件不是多用户数据库，不建议多台电脑同时写入同一个目录。

Portable EXE 目录不会自动生成配置文件、位置指针、初始化标记或数据文件夹。Windows 当前用户的应用数据目录中只保存一个 `storage-state.json`，用于记录所选数据路径和引导状态；它不包含 Key、正文、历史、音频或普通应用设置。

## GitHub 与安装包不包含用户数据

公开源码和构建配置只收录项目白名单文件，不收录 `Mytimes-TTS-Data`、自定义数据目录或旧 AppData。因此：

- 不包含用户已经填写的 API Key。
- 不包含 `settings.json`、`history.json`。
- 不包含参考音频、生成 WAV 或另存音频。
- 不包含个人目录、日志和签名凭据。
- 打包不会清空或修改当前电脑或网络目录中的 Key 与历史。

当前 Windows 应用包只收录运行所需的 `electron/`、`app/`、应用图标和 `package.json`。IndexTTS2 网关源码只进入公开源码包，不把 Python 环境、CUDA、模型或 checkpoints 打进 EXE。

## 如何清除数据

- 删除单条历史：同时删除应用保存的对应 WAV 副本。
- 清空历史：删除应用保存的全部历史记录和对应 WAV。
- 在设置中清空 API Key 并保存：移除应用保存的 Key。
- 删除 Portable EXE 不会自动删除所选数据目录或 Windows 用户目录中的路径索引。
- 删除 `storage-state.json` 只会使首次引导重新出现，不会删除业务数据。
- 如需彻底清除，在设置页确认当前数据目录后退出应用，再删除整个数据目录。

应用不会删除用户另存到其他目录的音频，也不会删除最初选择的声音样本。

## 声音复刻

声音可能涉及个人信息、人格权、著作权、合同和平台规则。只能上传本人声音，或已经取得明确、有效授权的声音。应用内的授权确认只是操作提醒，不能替代真实授权和法律审核。

不得使用本应用从事冒充、诈骗、骚扰、未经同意的公开传播或其他违法、侵权活动。

## API Key 安全

- 每位用户使用自己有权使用的 Key。
- 不要在截图、录屏、Issue、聊天、日志或 Git 仓库中公开 Key。
- Key 泄露后，应立即在对应服务控制台重置或删除，并检查调用记录。
- 统一提供额度时，应使用受控后端代理，不能把共享 Key 内置进公开 EXE、APK 或网页。

## 联系与变更

安全问题请使用本仓库的 GitHub Security Advisory 私下报告；不要在公开 Issue 中发布真实 Key、用户正文或声音样本。

未来若增加账号、服务器代理、遥测、崩溃上报、云同步或移动端权限，本说明必须在发布前同步更新。
