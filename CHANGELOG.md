# 更新记录

## 0.2.1

- 首次设置增加“公开版”和“MytimesXR 内部设置”选择；公开版为默认推荐。
- 公开版不会探测组织内部目录，只使用用户选择的本机或网络数据目录。
- 内部设置改用安装包之外的受控 JSON 配置文件；默认位置不可用时可手动选择。
- 修复设置页引擎类别缺少边框的问题，并统一引擎卡片、标签页、输入框和焦点状态的 Fluent 视觉。
- 生成页当前引擎改为中性边框与左侧选择条，避免整张卡片出现过亮描边。
- README、安装教程、隐私说明和安全文档统一改为面向公开用户。
- 发布包继续排除内部配置、API Key、用户设置、历史、参考声音和生成音频。

## 0.2.0 · 2026-07-14

- 新增 Xiaomi MiMo、火山引擎、GPT-SoVITS 与 IndexTTS2 四引擎切换。
- 将原 MiMo 请求迁移到统一 Provider 架构，波形、播放器、保存和历史继续复用。
- 设置页新增分引擎连接配置；MiMo、火山引擎和可选本地服务 Key 分别使用 Windows DPAPI 加密。
- 火山引擎加入 V1 HTTP 兼容合成、音色 ID、Cluster、语速、超时与费用确认。
- 引擎卡片只展示云端或本地类型，不再向普通用户显示“已接入”“接入中”等开发状态。
- 标题栏与侧栏品牌区统一使用当前无描边 Mytimes TTS 图标。
- GPT-SoVITS 接入官方 api_v2.py 的 /tts，支持参考音频、参考原文、语言与文本切分。
- IndexTTS2 新增独立 FastAPI 本地网关、文字情绪、八维情绪、随机采样和 FP16 启动说明。
- 本地模型、Python 环境、CUDA、checkpoints 与整合包不进入 GitHub 源码包或 Windows EXE。
- 统一使用去除外圈描边的波形 Logo，并从 SVG 生成 PNG 与多尺寸 ICO。
- 增加多引擎部署教程、隐私说明和持续更新的任务恢复文档。

## 0.1.7 · 2026-07-14

- 修复首次引导页左上角图标在打包后缺失的问题。
- “规则整理”保持纯本地运行，不调用任何大模型，不消耗 MiMo Token。
- AI 风格润色改用完全独立的 llama.cpp / OpenAI 兼容接口；默认关闭，绝不自动使用或回退到 Xiaomi MiMo API Key。
- 设置页增加独立润色地址、模型名、可选 API Key 和超时时间；独立 Key 继续使用 Windows DPAPI 加密。
- MiMo 连接测试增加费用提醒和二次确认；只保存设置不会产生 API 请求。
- 设置页增加“重新演示首次引导”，仅重置引导显示状态，不删除数据、设置、历史或 API Key。

## 0.1.6 · 2026-07-14

- Portable EXE 保持单文件，运行目录不再生成数据位置指针、初始化标记或默认数据文件夹。
- 设置、历史、生成音频和本机加密 Key 只写入首次选择的数据目录。
- Windows 用户目录只保存一个很小的目录索引，不包含 API Key、正文、历史、音频或普通应用设置。
- 首次选择后会自动记住数据位置；只有目录被移动、删除或 NAS 断开时才再次显示位置恢复引导。
- 本机备用位置改为“文档\\Mytimes-TTS-Data”，避免 EXE 和业务文件混放。
- 自动迁移 0.1.5 的 EXE 旁位置记录与初始化状态；只清理两个旧索引文件，不删除旧数据目录。

## 0.1.5 · 2026-07-14

- 新增只出现一次的初始化引导：介绍文字转语音流程、选择数据目录并检查配置与本机 API Key。
- 首次启动自动检测 `Y:\【软件插件】\Mytimes-TTS-Data`，发现共享配置时可一键使用。
- 没有 Y: 映射盘时提供选择其他 NAS/本机目录和使用 EXE 旁目录，不阻塞应用启动。
- 未检测到 `settings.json` 或本机加密 API Key 时，完成引导后直接进入设置页。
- 设置页增加“使用公司 NAS”快捷按钮，仍保留选择、打开和恢复默认目录操作。
- 初始化完成状态只保存在 EXE 旁的非敏感引导标记中，不进入 GitHub 或应用包。

## 0.1.4 · 2026-07-13

- 默认在 Portable EXE 旁创建 `Mytimes-TTS-Data`，统一保存设置、历史和生成音频。
- 设置页新增数据目录显示、选择、打开与恢复默认操作，支持本机磁盘和 NAS 映射目录。
- 首次启动新版时复制旧 AppData 数据到便携目录，保留旧文件作为回退，不自动删除。
- API Key 从通用设置中拆分为按电脑和 Windows 用户隔离的 DPAPI 加密文件，NAS 多电脑使用时互不覆盖。
- 切换目录时可选择使用目标目录现有数据，或复制当前数据并自动修正历史音频路径。
